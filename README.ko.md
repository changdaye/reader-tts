# reader-tts

## 언어: [中文](README.md) | [한국어](README.ko.md) | [English](README.en.md)

## 개요

reader-tts는 마크다운 파일을 음성으로 변환하는 가벼운 명령줄 도구입니다. 운영체제 내장 텍스트 음성 변환(TTS) 기능을 활용하여 실행 속도가 빠르고 사용하기 쉬운 인터페이스를 갖추고 터미널에서 직접 실행할 수 있습니다. 이 도구는 브라우저 응용 프로그램을 열지 않고 텍스트를 듣기만 원하는 사용자를 위해 특별히 설계되었습니다.

### 주요 기능:
- 효율적인 명령줄 인터페이스
- 속도 조절 기능 포함 ("2배 속도가 너무 느림" 문제를 해결하며 더 빠른 옵션 제공)
- 볼륨 제어
- 음성 선택
- 다국어 문서 지원
- 무음 모드 옵션
- 복잡한 마크다운 요소 지원 (헤더, 목록, 코드 블록 등)
- 통합 오류 처리

## 시스템 요구사항

- Node.js v14.0 이상
- 텍스트 음성 변환 기능을 포함한 운영체제
  - macOS: 내장된 `say` 명령에 대한 네이티브 지원
  - Linux:  `espeak` 혹은 `festival`(별도 설치 필요할 수 있음)
  - Windows: 시스템 오디오 프로그래밍 인터페이스 (SAPI)

## 설치

### 권장 설치 방법
가장 쉬운 설치 방법은 npm으로 글로벌 설치하는 것입니다:

```bash
npm install -g reader-tts
```

### 수동 설치
프로젝트를 수동으로 설치하시려면:

1. 로컬에 프로젝트 복제
```bash
git clone https://github.com/changdaye/reader-tts.git
cd reader-tts
```

2. 프로젝트 의존성 설치
```bash
npm install
```

3. 글로벌 연결
```bash
npm link
```

### 설치 확인
프로그램이 올바르게 설치되었는지 확인하십시오:

```bash
reader-tts --version
```

버전 번호를 볼 수 있다면 설치에 성공한 것입니다.

## 기능

- **지능형 텍스트 분석**: 마크다운 파일에서 자동으로 텍스트 내용만 추출하고 포맷은 제거하며 의미 있는 정보를 보존합니다.
- **고도로 맞춤화**: 속도, 볼륨, 화자 등을 포함한 음성 재생 다양한 매개변수 조정 가능
- **다양한 속도 지원**: 2배 초과 속도 지원 ("2배속이 느림" 요구 사항 해결), 선호도에 따라 속도 조절
- **여러 파일 형식 지원**: `.md` 및 `.markdown` 확장자 모두 지원
- **컨텍스트 메뉴 통합**: macOS에서 우클릭 컨텍스트 메뉴와 통합 지원
- **오류 처리**: 친절한 오류 알림 및 확인 정보 제공

## 사용법

### 기본 사용법
```bash
reader-tts <마크다운-파일-경로>
```

예시:
```bash
reader-tts ./article.md
```

### 자세한 명령 형식
```bash
reader-tts [options] [file]
```

옵션들:
- `file`: 읽을 마크다운 파일 경로를 지정하는 필수 매개변수
- `-s, --speed <speed>`: 읽기 속도 (기본: 1.0x)
- `-r, --rate <rate>`: 속도 값의 대체 매개변수 (기본: 1.0x)
- `-v, --voice <voice>`: 사용할 음성 (지정하지 않을 경우 시스템 기본값 사용)
- `--volume <volume>`: 볼륨 레벨 (0.0 ~ 1.0, 기본: 1.0)
- `--quiet`: 정보 출력 숨기기
- `--install-right-click`: macOS에 우클릭 옵션 설치
- `-h, --help`: 도움말 정보 출력
- `-V, --version`: 도구 버전 표시

## 매개변수 세부정보

각 매개변수는 사용자의 경험을 극대화하기 위해 명확한 기능과 용도를 가지고 있습니다.

### 속도 조절

`--speed` 또는 `-s`: 기본값은 `1.0x`로 일반 말하는 속도를 나타냅니다.
- `1.0` 미만의 값은 속도를 저해: 예시로 `0.8`은 0.8x 속도를 나타냅니다
- `1.0` 초과의 값은 속도를 향상: `1.5`는 1.5x 속도입니다
- **"2x 속도가 느림" 요구 사항에 따라 `--speed 3.0` 또는 그 이상으로 설정할 수 있습니다 (예: `--speed 5.0`)**

예시:
```bash
reader-tts README.md --speed 3.0  # 3.0x 속도로 읽기
```

### 음성 설정
`--voice` 또는 `-v`: 운영 체제에서 이용 가능한 특정 음성을 선택할 수 있도록 합니다. 각 운영 체제는 서로 다른 음성 라이브러리를 제공합니다.

가능한 음성을 확인하는 것은 OS에 따라 다릅니다. macOS에서는 "시스템 환경설정" > "접근성" > "음성"을 통해 음성을 확인하거나 추가할 수 있습니다.

```bash
reader-tts README.md --voice "Alex"  # Alex 음성으로 읽기
```

### 볼륨 레벨
`--volume`: `0.0`(무음)에서 `1.0`(최대 볼륨) 사이의 볼륨 설정, 기본값은 `1.0`.
```bash
reader-tts README.md --volume 0.7  # 70% 볼륨으로 읽기
```

### 무음 모드
`--quiet`: 이 모드에서는 도구는 상태 정보를 출력하지 않고 바로 음성 재생을 진행합니다.
```bash
reader-tts README.md --quiet  # 무음 모드로 직접 정보 없이 재생
```

## 사용 예시

이 도구의 유연성을 보여주는 평소 쓰이는 명령어 예시입니다.

### 기본 예시
가장 간단한 읽기 명령어로 기본 읽기 속도 사용:
```bash
reader-tts document.md
```

### 다양한 속도 예시
빠르게 개요를 얻기 위해 속도 향상 또는 익숙한 컨텐츠 절약을 위해 시간 절약:
```bash
# 약간 더 빠르게 하기
reader-tts document.md --speed 1.5

# "2x가 느림" 요구 사항 충족 (2.0x보다 빠름)
reader-tts document.md --speed 3.5

# 훨씬 빠른 속도 (반복적이거나 암기된 내용 듣기 위함)
reader-tts document.md --speed 5.0
```

### 볼륨 및 속도 결합 예시
여러 매개변수 동시에 조정:
```bash
reader-tts important-notes.md --speed 2.0 --volume 0.6
```

### 완전히 구성화된 예시
개인화 경험을 위한 여러 매개변수 설정:
```bash
reader-tts thesis.md --speed 1.8 --voice "Victoria" --volume 0.8 --quiet
```

## macOS 통합

이 도구는 macOS 시스템 통합 해결책을 제공하여 우클릭을 통해 쉬운 접근이 가능하게 합니다. 다음 명령어로 함수를 활성화하십시오:

```bash
reader-tts --install-right-click
```

이 명령어는 macOS 마우스 오른쪽 클릭 메뉴 설치에 대한 구체 지시사항을 보여줍니다. 상세 단계는 다음과 같습니다:

1. **오토메이터를 사용한 액션 빠른 만들기**:
   1. macOS에 기본 탑재된 "오토메이터" 애플리케이션 열기
   2. "퀵 액션" 템플릿 선택하고 새 오토메이션 만들기
   3. 오른쪽 "라이브러리"에서 "쉘 스크립트 실행"을 찾아 워크스페이스에 끌기
   4. 이 스크립트 설정:
      - 쉘 선택기: `/bin/bash`
      - 입력 전달: `인자 형태`
      - 다음 커맨드를 스크립트 내용으로 입력:

      ```bash
      for f in "$@"; do reader-tts "$f"; done
      ```

      5. 작동 흐름 저장 후 "TTS로 읽기"와 같은 의미 있는 이름 부여

2. **스크립트 편집기 사용한 대안 계획**(고급 자동화가 필요 없는 경우):
   1. 내장 "스크립트 편집기" 애플리케이션 열기
   2. 다음 애플스크립트 붙여넣기:

   ```applescript
   on run {input, parameters}
     repeat with aFile in input
       set filePath to POSIX path of (aFile as string)
       do shell script "reader-tts \"" & filePath & "\""
     end repeat
     return input
   end run
   ```

   3. "응용 프로그램"으로 저장
   4. `~/Library/Services/`로 이동
   5. 시스템 환경설정 > 키보드 > 단축키 > 서비스에서 활성화

### 통합 클릭 옵션 사용
설치에 성공하면 지원되는 마크다운 파일에서 마우스 오른쪽 단추를 눌러 해당 서비스를 선택하여 재생을 빠르게 할 수 있습니다.

## 문제 해결

문제가 발생하면 다음 해결 방안을 참조하십시오:

- **`reader-tts` 명령어를 찾을 수 없음**
  - 글로벌 설치를 올바르게 했는지 확인
  - npm 글로벌 패키지 디렉토리가 `$PATH`에 있는지 확인
  - 재설치 시도: `npm install -g reader-tts`

- **출력된 오디오 없음**
  - 시스템에 TTS 기능이 지원되는지 확인
  - 스피커나 헤드폰이 연결되어 있는지 확인
  - 시스템 내장 TTS 사용 가능 여부 테스트 (macOS: `say hello`, Linux: `espeak hello`)

- **특정 음성 사용 불가**
  - `--voice` 매개변수 사용 전 입력된 음성이 시스템에서 지원되는지 확인
  - 일부 시스템에서는 음성 팩을 추가로 다운로드해야 할 수도 있음

- **비어 있는 파일이거나 잘못된 파일 형식**
  - 제공된 파일이 존재하고 비어 있지 않은지 확인
  - `.md` 또는 `.markdown` 확장자인지 확인

## 기술적 세부정보

이 프로젝트는 다음 모듈들로 구성되어 있습니다:

- `bin/reader-tts`: 명령줄 진입점으로 매개변수 구문 분석 담당
- `src/index.js`: 핵심 기능 모듈로 파일 분석 및 오디오 합성 담당
- `src/context-menu.js`: macOS 통합을 특별히 처리하는 보조 기능
- `package.json`: 종속 항목 및 명령어 진입점 정의

TTS 엔진은 [`say`](https://github.com/Marak/say.js) 오픈 소스 라이브러리를 기반으로 하며 각 시스템의 하위 TTS 기능을 래핑합니다.
- macOS: 내장된 `say` 명령 사용
- Linux: `festival` 또는 `espeak` 라이브러리 접근
- Windows: SAPI 사용

마크다운 분석에는 대부분의 표준 및 비표준 마크다운 구문을 처리할 수 있는 고성능 `markdown-it` 라이브러리 사용.
